02 · The shift

Token economy → agent economy

S1 / 2023

The prompt

One call, one answer. Intelligence as a vending machine — no memory, no stakes, no economics beyond the invoice.

S2 / TOKEN ECONOMY

The token router

OpenRouter’s world: arbitrage price per million tokens, pick the cheapest capable model, per request. Every call is a stranger — the router never learns whether the answer was right.

S3 / WORKFLOWS

Chained calls

Pipelines and glue code. Steps get cheap, but nobody owns the outcome — one silent upstream error poisons everything downstream.

S4 / BACKGROUND AGENTS

The unsupervised swarm

Agents run for hours without a human in the loop. Inference demand multiplies 10–100×. Spend explodes — with no measure of verified value per dollar.

S5 / AGENT ECONOMY

The allocator

Digital workers with identities, budgets, autonomy and P&L. Capital, verification and permissions routed by verified marginal value — not token price. This is the layer above every router.

03 · Decomposition & routing

One task, many minds — priced per step

Decompose a comprehensive task and most steps stop needing a frontier model. Reading a 5-page PDF routes to an open-source model for cents; the deep-research core routes automatically to GPT 5.6 or Fable 5 — with an independent verifier attached wherever the blast radius is large. Every step is calibrated on the cost × accuracy × speed triangle.

04 · Research

From one-shot models to durable background agents

Built from a finance-desk testbed: a one-shot agent that turns a ticker into a full investment model — historicals, forecasts, DCF, three statements, a decision. Making it durable exposed three capabilities that “keep a log” cannot provide.

R·01

Causal execution, not chronological logging

Every claim links to its evidence, assumptions, dependencies, provenance, and downstream blast radius — so the system knows exactly which errors can propagate through the workflow.

execution ledger
R·02

Risk-weighted verification

Stronger models, independent evaluators, and human review flow only to decisions whose failure would invalidate the broader task. Low-risk steps get cheap models, caches, and minimal checks.

verification budget
R·03

Durable learning & co-evolution

Execution state survives interruptions and context resets. Critical checkpoints are selectively revalidated as prompts, tools, guardrails, or the underlying models change.

05 · System

A portfolio manager for digital workers

Enterprises are beginning to run two workforces — human and digital. Observability shows what an agent did. It does not decide which agents should receive more work, money, compute, memory, or autonomy. That is an allocation problem, and it needs a CFO.

S·01

Agent capital allocation

Tasks, inference spend, and compute flow to agents producing the highest verified marginal value — and the system says when a bigger budget beats further optimization.

cost / verified outcome
S·02

Earned autonomy

Permissions, budget, and independence expand or contract on accumulated performance — probation to supervised to trusted to autonomous — never on static settings.

performance-gated permissions
S·03

Decision continuity

What was decided, why, which alternatives were rejected, and what happened next — preserved even when the model or the agent behind it is replaced. Every action is a small experiment.

institutional memory
